# CLAUDE.md — Project guide for Claude Code

## Project overview

CFC Singapore (Couples for Christ) website. Static HTML site deployed on GitHub Pages.

- **Live site:** https://bordy20.github.io/cfc-sg-portal/
- **Repo:** https://github.com/bordy20/cfc-sg-portal
- **Hosting:** GitHub Pages (auto-deploys on push to `main` via `.github/workflows/pages.yml`)

## Site architecture (two layers)

### 1. Landing page (public-facing hero page)
- `index.html` — single-page design with hero, about, ministries, events, stories, footer
- `assets/style.css` — blue/indigo + gold palette, Lora + Nunito fonts, responsive
- `assets/main.js` — nav toggle, IntersectionObserver fade animations, scroll shadow
- Uses embedded SVG art (no external images)
- Google Fonts: Lora (serif headings) + Nunito (sans-serif body)

### 2. Content pages (181 internal pages)
- 11 top-level sections: `home/`, `responsibility/`, `holiness/`, `household-and-home/`, `teachings/`, `tithes/`, `chapter/`, `evangelization-and-clp/`, `ancop/`, `mission-centre/`, `zoom-tutorials/`
- Each page is `<section-slug>/<page-slug>/index.html`
- Content pages use their own theme (Fraunces + Inter fonts, sidebar navigation, breadcrumbs)
- Generated by `build-site.js` from a sitemap defined in that file

## Key files

| File | Purpose | Safe to edit directly? |
|------|---------|----------------------|
| `index.html` | Landing page | Yes — this is hand-crafted, not generated |
| `assets/style.css` | Landing page CSS | Yes — but careful, affects entire landing page |
| `assets/main.js` | Landing page JS | Yes |
| `build-site.js` | Content page generator | Yes — edit the `sitemap` array to add/remove/rename content pages, then run `node build-site.js` |
| `.github/workflows/pages.yml` | Deployment workflow | Rarely needs changes |
| Content pages (`home/*/index.html`, etc.) | Generated content | Can edit directly, but will be overwritten if `build-site.js` is re-run |

## Common tasks

### Update landing page content
Edit `index.html` directly. Key sections are marked with HTML comments:
- `<!-- ===== HERO ===== -->` — main heading, description, CTA buttons
- `<!-- ===== WELCOME STRIP ===== -->` — four pillars row
- `<!-- ===== ABOUT ===== -->` — "Who We Are" section
- `<!-- ===== HOLY FAMILY ===== -->` — dark blue section
- `<!-- ===== MINISTRIES ===== -->` — 6 ministry cards with links
- `<!-- ===== EVENTS ===== -->` — upcoming events (dates, locations, descriptions)
- `<!-- ===== COMMUNITY STORIES ===== -->` — testimonials
- `<!-- ===== FOOTER ===== -->` — links, scripture, copyright

### Add a new content page
1. Edit `build-site.js` — add entry to the `sitemap` array under the right parent
2. Run `node build-site.js` to regenerate all pages
3. The new page appears with correct sidebar, breadcrumbs, and navigation
4. Optionally add a link from the landing page if it's a key page

### Delete a content page
1. Edit `build-site.js` — remove the entry from the `sitemap` array
2. Run `node build-site.js` to regenerate
3. Delete the leftover folder: `rm -rf <section>/<slug>/`
4. Check if the landing page (`index.html`) links to it — remove those links
5. Commit and push

### Add/remove a whole section
Same as above but at the top level of the `sitemap` array. Also update the `GENERATED` array at the top of `build-site.js` if adding a new section slug.

### Update events on landing page
Find `<!-- ===== EVENTS ===== -->` in `index.html`. Each event has:
- `.ev-s-date` or `.ev-cal` — month and day
- `h3`/`h4` — event name
- `p` — description
- `.ev-where` — location with pin emoji
- `href` — link to detail page

### Update ministry card links
Each `.min-card` in the ministries section has an `<a href="..." class="min-cta">` pointing to a content page. Update the `href` to change where "Learn More" goes.

## Deployment

Push to `main` triggers automatic deployment via GitHub Actions. Site updates in ~1 minute.

```bash
git add .
git commit -m "Description of changes"
git push origin main
```

## Build system

The site generator is optional — only needed for structural changes to the 181 content pages:

```bash
node build-site.js
```

This regenerates all content pages from the sitemap. It does NOT touch:
- `CLAUDE.md`, `README.md`, `DEPLOY.md`, `EDITING.md`, `DIRECTORY-GUIDE.md`
- `.git/`, `.nojekyll`, `.github/`
- Any files outside the 11 section folders

**Warning:** The generator WILL overwrite `assets/style.css`, `assets/main.js`, and the root `index.html` with its own versions. If you've customized the landing page (which we have), do NOT run the generator without first backing up or modifying `build-site.js` to skip these files.

## Git workflow

- Main branch: `main`
- All changes go directly to `main` (no PR required for content updates)
- Commit messages should describe what changed: "Update CLP event date to August 2026"
- Always push after committing — the live site only updates on push
